项目概述

UnityCoopPuppetGame 是一款基于 Unity 引擎的双人协作木偶游戏。两名玩家通过锁链相连，在精心设计的关卡中携手解谜、共闯险境。游戏采用纯代码生成几何体模型呈现实木木偶角色，配合青橙电影级调色风格，打造独特的视觉体验。

核心特性

  • 渲染管线 — Unity 6.5 URP（Universal Render Pipeline），支持高质量后处理效果与移动端优化渲染
  • 电影级调色 — 青橙互补色（Teal & Orange）电影调色方案，营造沉浸式的视觉氛围
  • 实时联机 — 基于 Photon Fusion 2 的状态同步框架，公网环境下低延迟双人协作
  • 锁链协作机制 — 双人通过物理锁链连接，移动距离受限，需密切配合推进关卡
  • 木偶角色系统 — 纯代码生成几何体组合模型，无需外部美术资产即可构建完整的木偶角色
  • 跨平台互通 — 支持 Android 与 iOS 公网联机互通，内置 NAT 穿透与中继服务器回退机制

核心联机

游戏联机系统基于 Photon Fusion 2 构建，采用状态同步（State Authority）模式，提供自动连接、房间管理和网络生命周期管理。整体网络架构分为多个职责单一的模块，便于维护和扩展。

NetworkManager.cs

网络管理总入口，统筹所有网络模块的生命周期。负责初始化各子管理器并协调联机流程。

ConnectionManager.cs

全自动连接管理器，处理 Photon 服务器的连接/断开/重连逻辑，内置断线重连和错误恢复机制。

GameLauncher.cs

游戏启动器，管理从主菜单到游戏场景的完整启动流程，包含房间创建、匹配和场景加载。

FusionNetworkManager.cs

Fusion 网络核心管理器，封装 Fusion Runner 的启动、关闭以及网络事件回调处理。

FusionPlayerSpawner.cs

网络玩家生成器，根据连接状态自动实例化或销毁玩家预制体，支持延迟加载。

PhotonAppIdConfig.cs

Photon App ID 配置组件，提供 Inspector 面板直接填写 App ID，避免硬编码密钥。

房间机制

  • 运行后 ConnectionManager 自动连接 Photon Cloud，无需手动操作
  • 若不存在可加入的房间，GameLauncher 自动创建新房间并等待另一名玩家加入
  • 若已存在可加入房间，自动加入并同步玩家状态
  • 房间支持 2 人协作模式，可通过 FusionSettings 修改房间参数

玩家同步

玩家同步系统负责将本地玩家的状态（位置、旋转、动画参数等）实时同步到远程客户端。同时支持 Fusion 网络版和本地调试版的切换。

PlayerSync.cs

核心同步组件，挂载在玩家预制体上。使用 Fusion 的 [Networked] 属性同步变换数据，处理插值与预测。

PlayerMovement.cs

移动控制器，处理玩家输入到角色移动的映射。支持键盘、触屏等不同输入源，内置锁链约束。

CharacterControllerMobile.cs

移动端专用控制器，针对触屏操作优化移动手感，包含虚拟摇杆和滑动方向检测。

PuppetBuilder.cs

木偶模型生成器，运行时通过代码生成几何体组合（身体、四肢、关节）构建完整的木偶角色模型。

PlayerController.cs

玩家高层控制器，协调移动、交互、动画状态机等子系统。

NetworkPlayer.cs

网络玩家标识组件，区分本地/远程玩家，管理玩家角色在网络上分配的权限。

触摸控制

移动端触摸控制系统提供直观的操作体验，包含动态虚拟摇杆、动作按钮、技能按钮以及统一的输入管理器，支持多点触控操作。

DynamicJoystick.cs

动态虚拟摇杆，触摸屏幕任意位置生成摇杆，松手后自动消失。支持灵敏度调节和方向死区配置。

MobileActionButton.cs

通用动作按钮，用于跳跃、拾取、互动等基础操作。支持长按与短按识别，可自定义图标和回调。

SkillButton.cs

技能按钮组件，支持技能冷却计时、释放效果触发和 MP 消耗逻辑，可扩展为多种技能类型。

MobileInputManager.cs

移动端输入总管理器，统一收集摇杆和按钮的输入信号，转换为标准化的输入数据供 PlayerMovement 消费。

输入处理流程

  • DynamicJoystick 输出方向向量 → MobileInputManager 汇总
  • MobileActionButton / SkillButton 发送动作事件 → MobileInputManager 转发
  • PlayerMovement 读取 MobileInputManager 数据执行移动和动作

跨平台

项目支持 Android 与 iOS 双平台公网联机互通。通过 Photon Cloud 的 NAT 穿透和中继服务器技术，确保不同网络环境下的玩家都能稳定连接。

CrossPlatformConfig.cs

  • 跨平台配置中心，管理平台差异化的参数（屏幕适配、输入类型、图形质量等）
  • 运行时自动检测当前平台并加载对应配置
  • 支持在 Inspector 中配置各平台参数预设

公网互通方案

  • Photon Fusion 2 内置 NAT 穿透（STUN），优先建立 P2P 直连以获得最低延迟
  • 当 P2P 不可用时，自动回退到 Photon Relay（中继服务器），确保连接成功率
  • Android 与 iOS 客户端共享同一 Photon App ID，使用相同的 Region 配置
  • 网络延迟自适应：Fusion 的 Tick System 根据网络条件动态调整同步频率

挂载步骤

以下是将网络系统挂载到 Unity 场景中的详细步骤。按照顺序操作即可快速搭建联机框架。

  1. 在场景 Hierarchy 中右键 → Create Empty，将空物体命名为 NetworkManager
  2. 选中 NetworkManager 物体，在 Inspector 中点击 Add Component，搜索并挂载 NetworkManager.cs
  3. 在 Inspector 中找到 Photon App Id 字段，填入你在 Photon Dashboard 申请的 App ID（可使用 PhotonAppIdConfig 组件管理）
  4. 在 Project 窗口中创建玩家预制体（右键 → Create → Prefab），将 PlayerSync、PlayerMovement、PuppetBuilder 等组件挂载到预制体根节点
  5. 选回 NetworkManager 物体，在 Inspector 的 Player Prefab 槽位中拖入刚才创建的玩家预制体
  6. 按下 Play 运行，ConnectionManager 将自动连接 Photon Cloud，GameLauncher 自动创建或加入房间

打包发布

项目内置 Editor 构建脚本，可一键执行 Android APK 打包流程。以下为手动配置和打包的详细说明。

Android 打包配置

  • 菜单 File → Build Settings，将 MainMenu.unity 和 MainGame.unity 添加到 Scenes In Build 列表
  • 确保 MainMenu.unity 位于索引 0（首个启动场景）
  • Platform 选择 Android，点击 Switch Platform

Player Settings 配置项

  • Company Name: 填写你的公司或团队名称
  • Product Name: UnityCoopPuppetGame
  • Version: 2.0.0
  • Bundle Identifier: com.yourcompany.unitycooppuppetgame
  • Minimum API Level: Android 8.0 (API Level 26)
  • Target API Level: Android 14 (API Level 34)
  • Scripting Backend: IL2CPP（推荐用于发布版本）
  • Target Architectures: ARM64
  • Install Location: Automatic

APK 构建

  • 菜单 File → Build Settings → Build（或使用 BuildScript.cs Editor 脚本一键构建）
  • 构建完成后在 Build/Android 目录找到生成的 APK 文件

签名配置

  • 菜单 Project Settings → Player → Android → Publishing Settings → Keystore Manager
  • 使用已有的 keystore 或点击 Create 创建新的签名文件
  • 填写 Keystore 密码、Key Alias 和 Key Alias 密码
  • 确保 Custom Keystore 已勾选并指向正确的 .keystore 文件
